Overview

The Help Editor is a powerful tool for managing the application's help documentation. It provides a professional editing environment with real-time preview, version control, and multiple template options.

ℹ️
Admin Access Required
Only administrators can access the Help Editor to maintain documentation quality and security.

Getting Started

Access the Editor
Navigate to Admin Dashboard → Help Editor or use the direct link in the admin menu.
Browse Files
Use the searchable file list on the left to find help files. Click on any file to open it in the editor.
Edit & Preview
Use the split-screen interface to edit HTML on the left and see live preview on the right.

Editing Files

CodeMirror Editor
Professional HTML editor with syntax highlighting, bracket matching, and auto-completion.
Real-time Preview
See your changes instantly in the preview panel with proper styling applied.
Keyboard Shortcuts
Use Ctrl+S (Cmd+S) to save and Ctrl+F (Cmd+F) to find text in the editor.
💡 Pro Tip
The preview updates automatically as you type, with a 500ms delay to prevent performance issues.

Creating New Files

Quick Recipe: Create New Help File
⏱️ 2 minutes
  1. 1
    Click the "New File" button next to the search bar
  2. 2
    Enter a filename (lowercase, numbers, underscores only)
  3. 3
    Choose a template that matches your content type
  4. 4
    Add title and description, then click "Create File"
⚠️
Filename Requirements
Filenames must contain only lowercase letters, numbers, and underscores. The .html extension is added automatically.

Version Management

Every edit creates a new version to preserve the editing history and allow rollbacks if needed.

Automatic Versioning
When you save a file, it creates a new version with format: filename_v2_username_YYYYMMDD.html
View Versions
Click the arrow next to files that have versions to expand and see all available versions.
Original Preserved
The original file remains unchanged, allowing you to always return to the base version.

Available Templates

🗒️ Blank

📖 Standard Help
Complete help page with navigation, sections, and quick actions.
🎯 Feature Documentation
Template for documenting specific features with alerts and steps.
📚 Tutorial/Guide
Step-by-step tutorial format with prerequisites and troubleshooting.

Best Practices

✨ Writing Guidelines
  • Use clear, concise language that users can easily understand
  • Include screenshots or examples when explaining complex features
  • Structure content with proper headings and sections
  • Test all links and ensure they work correctly
🔧 Technical Tips
  • Always preview your changes before saving
  • Use the existing CSS classes for consistent styling
  • Keep HTML semantic and accessible
  • Save frequently to create regular checkpoints
🎯 Content Strategy
  • Start with user goals and work backwards
  • Include quick actions to help users get started
  • Provide troubleshooting for common issues
  • Update help content when features change
