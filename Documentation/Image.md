1. Requirements & User Stories
1.1. Core Goal
To empower users to generate, manage, and associate high-quality, context-aware images with their primary creative assets (Characters, Locations, LoreItems, and Scenes).
1.2. Key Features
Provider Agnosticism: The system must be designed to support multiple AI image generation models (e.g., DALL-E 3, RunPod/Stable Diffusion) through a "provider" pattern.
Config-Driven Provider: The active image generation provider will be determined by a single setting in the .env file (ACTIVE_IMAGE_PROVIDER).
Standardized Prompts: Characters, Locations, and LoreItems will each have a dedicated image_prompt_definition field to store their canonical visual description.
Contextual Scene Prompts: The system will intelligently pre-fill a prompt for Scene illustrations by combining the scene's action and mood with the standardized prompts of the characters and locations involved.
Global Style Control: Users can create and save "Image Style" prompts (e.g., "photorealistic, cinematic lighting") in the Prompt Library and apply them to any generation request to maintain artistic consistency.
Asynchronous Generation: Image generation will be a non-blocking background job. The user will be notified via a toast message upon completion.
Image Versioning: The system will store every generated image, allowing users to maintain a gallery of versions for each asset and choose a "current" one.
Usage Control: A simple cooldown mechanism will be implemented to prevent rapid, successive generation requests for the same element, helping to manage costs.

2.2. New & Modified Service Components
app/services/image_providers/ (New Package):
base_provider.py: Defines the abstract BaseImageProvider class and the ImageGenerationResult Pydantic model, creating a strict contract for all providers.
dalle3_provider.py: A concrete provider for making direct API calls to Azure DALL-E 3.
runpod_provider.py: A concrete provider implementing the run -> poll -> download logic for RunPod/Stable Diffusion.
app/services/image_service.py (New File):
Acts as a factory. Reads settings.ACTIVE_IMAGE_PROVIDER.
Exposes a single function: generate_image_with_active_provider(prompt, user_id).
app/routers/image_generation.py (New File):
The public API endpoint.
Handles request validation, authentication, and job creation.
Orchestrates the background task that uses the ImageService.
3. Database Changes
3.1. New Table: generated_images
This table will store metadata for every generated image, enabling versioning.
Column Name	Data Type	Constraints / Notes
id	INTEGER	Primary Key, Auto-incrementing
image_uuid	UUID	Required, Indexed. A unique ID for the file itself, generated by the application.
blob_path	VARCHAR(1024)	Required. Full path in blob storage, e.g., worlds/1/characters/5/uuid.png.
prompt	TEXT	Required. The prompt sent to the AI for generation.
revised_prompt	TEXT	Nullable. The revised prompt returned by the AI (especially from DALL-E 3).
element_type	VARCHAR(50)	Required, Indexed. e.g., 'character', 'location', 'scene'.
associated_element_id	INTEGER	Required, Indexed. The ID of the Character, Location, or Scene.
user_id	INTEGER	Required, Indexed. Foreign Key to users.id.
created_at	TIMESTAMPTZ	Required.
3.2. Modifications to Existing Tables
The following nullable Foreign Key columns will be added to enable a "current" or "featured" image for each element.
characters table:
Add current_image_id: INTEGER, FK to generated_images.id (ON DELETE SET NULL)
locations table:
Add current_image_id: INTEGER, FK to generated_images.id (ON DELETE SET NULL)
lore_items table:
Add current_image_id: INTEGER, FK to generated_images.id (ON DELETE SET NULL)
scenes table:
Add current_image_id: INTEGER, FK to generated_images.id (ON DELETE SET NULL)
3.3. Modifications to Model prompt.py
Add IMAGE_STYLE to the PromptTypeEnum.
4. Configuration Changes (.env and app/core/config.py)
The following variables will be added:
ACTIVE_IMAGE_PROVIDER: "DALLE3" (or "RUNPOD")
DEFAULT_IMAGE_SIZE: "1024x1024"
AZURE_STORAGE_CONTAINER_NAME_FOR_GENERATED_IMAGES: "generated-images"
RUNPOD_API_KEY: your_runpod_api_key (for future use)
RUNPOD_API_ENDPOINT_ID: your_runpod_endpoint_id (for future use)
5. UI Element Design & Placement
5.1. Character, Location, & Lore Item Forms (..._form.html)
Location: Inside the right-hand sidebar.
New Section: "Appearance & Image Generation".
Elements:
Textarea (image_prompt_definition): For the user to write the base visual description.
Dropdown (image_style_select): Populated with prompts of type IMAGE_STYLE from the Prompt Library. Labeled "Select Image Style".
Button (generate-image-btn): Labeled "Generate Image". It will be disabled during generation and cooldown.
Image Display Area: A container showing the current_image_id for the element. Displays a spinner while loading.
Image Gallery / Version History: A new section below the main image that displays thumbnails of all other images in the generated_images table associated with this element. Each thumbnail will have a "Set as Current" button.
5.2. Scene Editor (scene_editor_ui.html)
Location: A new tab in the right-hand AI pane labeled "Illustration Studio".
Elements:
Textarea (scene_image_prompt): A large textarea, pre-filled by JavaScript with the intelligently constructed prompt. The user can edit this freely.
Dropdown (image_style_select): Same as above, for applying a global style.
Button (generate-scene-image-btn): Labeled "Generate Scene Illustration".
Image Display & Gallery: Same as for world elements, showing the current_illustration_id and a gallery of other versions for that scene.
5.3. Prompt Library (prompt_form.html)
The "Prompt Type" dropdown will now include the "Image Style" option.
