## Documentation Standards

### Formatting Guidelines
- Use Markdown format for all documentation files
- Include table of contents for documents longer than 500 words
- Use consistent heading hierarchy (H1 for main sections, H2 for subsections)
- Include code examples in appropriate language-specific code blocks
- Use Mermaid syntax for all architectural diagrams

### Diagram Standards
- All diagrams should be created using Mermaid syntax
- Include both source `.mmd` files and exported formats in the diagrams directory
- Use consistent color schemes and styling across diagrams
- Include descriptive titles and legends where appropriate

### Content Guidelines
- Write for multiple audiences (developers, architects, operations)
- Include practical examples and code snippets
- Reference specific files and line numbers where relevant
- Keep documentation current with code changes
- Include links to external resources and dependencies

### Review Process
- All documentation changes should be reviewed by technical leads
- Validate accuracy against current implementation
- Test all procedures and setup instructions
- Update version information and last modified dates
