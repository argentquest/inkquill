Adopting the new Family Home design

The redesign in Care Circle Family.html takes the broadsheet vocabulary from the newsletter and applies it to app/care-circle-family/page.tsx. The route, data, and nav stay the same — what changes is the page composition: a broadsheet masthead, a photo-led "Today" hero, a two-column body, provenance pills, and an activity ledger styled as a newspaper column.

What stays · what changes

StaysChanges
The route /care-circle-family, the nav link list, the icon components in care-circle-feature-icons.tsx, the platform shell, the auth/membership guards. The page-level layout in app/care-circle-family/page.tsx, the section header style, and the card markup — everything visual below the global topbar.

Anatomy of the new page

Masthead
Centered broadsheet block: small eyebrow, large display title ("The Silver Family Circle"), italic foot line ("Caring for Rose Ellis · weather · N contributors"). Triple-rule (top single, bottom double).
Greeting strip
Single-line welcome with caregiver first name in italic ember; right-aligned meta about next session and last delivery.
Today hero
Photo-led 2-column panel. Left: latest queued lead photo with a provenance pill overlay + italic caption. Right: eyebrow, headline, lede, then a 3-up metrics rule.
Section header
Heavy bottom rule, display-serif title with an ember dot, right-aligned "Section A · Manage all →" meta.
Care & Circle grid
2×3 nav cards (was 3×2). First card is filled --accent (ink-900 background) to anchor the eye. Each card carries a small live count and an arrow CTA. Members card shows actual avatars + Invite button.
Side stack
Voice message in an ember gradient card (mirrors newsletter), On-this-day pull-quote, Birthdays card, Garden almanac card. All carry provenance pills.
Community ledger
Activity surfaces as a newspaper-style ledger: time · entry · tag rows separated by dashed rules. Two trailing cards for forum thread and help guide.
Colophon
Italic closer above a triple-rule, mirroring the newsletter footer.

The plan, in five steps

Step 1

Add a shared FamilyMasthead component

Create frontendv1/components/care-circle-family/family-masthead.tsx. Props: { familyName, patientName, weather, contributorsCount, edition }. Reuse this on every Family page so the broadsheet identity carries through.

// family-masthead.tsx
export function FamilyMasthead({ familyName, patientName, weather, contributorsCount, edition }) {
  return (
    <header className="cc-mast">
      <span className="cc-mast__corner cc-mast__corner--l">{edition}</span>
      <span className="cc-mast__corner cc-mast__corner--r">{formatDate(new Date())}</span>
      <div className="cc-mast__eyebrow">The Family Daily</div>
      <h1 className="cc-mast__title">The {familyName} Family Circle</h1>
      <p className="cc-mast__foot">
        Caring for <strong>{patientName}</strong> · {weather} · {contributorsCount} contributors today
      </p>
    </header>
  );
}

Tailwind classes or a co-located family-masthead.module.css are both fine — the rules are short. The mockup uses plain CSS in .cc-mast* for clarity.

Step 2

Add a ProvBadge primitive — single source of truth

The same component you'd add to the email templates. Three kinds: family / curated / auto, each with its own inline SVG glyph. Keep it tiny:

// components/care-circle-family/prov-badge.tsx
const ICONS = {
  family:  <path d="M12 20s-7-4.5-7-10a4 4 0 0 1 7-2.65A4 4 0 0 1 19 10c0 5.5-7 10-7 10z" />,
  curated: (<><path d="M3 5h7a2 2 0 0 1 2 2v13" /><path d="M21 5h-7a2 2 0 0 0-2 2v13" /><path d="M3 5v14h18V5" /></>),
  auto:    (<><path d="M12 4v2M12 18v2M4 12h2M18 12h2" /><circle cx="12" cy="12" r="3" /></>),
};
export function ProvBadge({ kind = "auto", children }) {
  return (
    <span className={`prov prov--${kind}`}>
      <svg width="12" height="12" viewBox="0 0 24 24" fill="none" stroke="currentColor" strokeWidth="1.8">{ICONS[kind]}</svg>
      {children}
    </span>
  );
}

Use this in the email templates too (Jinja macro mirrors the same SVG). One visual vocabulary across web + email.

Step 3

Replace the page body in app/care-circle-family/page.tsx

The current page maps a flat nav array to identical tile cards. Replace it with a structured layout that uses the same array but composes them into the redesign:

// app/care-circle-family/page.tsx — sketch
const careCircleNav = nav.filter(n => n.group !== "community");
const communityNav  = nav.filter(n => n.group === "community");

return (
  <main className="cc-page">
    <Breadcrumbs trail={[{label:"Home"}, {label:"Care Circle Family", current:true}]} />

    <FamilyMasthead familyName={family.name} patientName={primaryPatient.name}
                    weather={today.weather} contributorsCount={today.contributors}
                    edition={`Vol. III · No. ${edition.number}`} />

    <GreetingStrip caregiverName={user.firstName} nextSessionIn={today.nextSessionIn}

    <div className="cc-cols">
      <div className="cc-grid">
        {careCircleNav.map((item, i) => (
          <NavCard key={item.href} accent={i === 0} {...item} count={counts[item.key]} />
        ))}
      </div>
      <aside className="cc-side">
        <VoiceMessageCard message={today.voiceMessage} />
        <OnThisDayCard event={today.onThisDay} />
        <BirthdaysCard items={today.birthdays} />
        <GardenAlmanacCard tip={today.almanac} />

None of these need to be heavy components — most are 10–20 lines. Co-locate them under components/care-circle-family/family-home/.

Step 4

Wire the data the new layout needs

The redesign surfaces a few new live values. They all already exist server-side — just need a single endpoint or composed query to deliver them in one round-trip:

FieldWhere it comes from
edition.number & lastDeliveredAtcare_circle_newsletter task results / delivery service
today.contributorscount of distinct family submissions in the last 24h
today.leadPhotomost recent media upload tagged for tomorrow's edition
today.voiceMessagemost recent audio submission (already in family contributions)
today.onThisDay / almanacprovider results from this_day_history / garden_almanac for today
today.birthdaysexisting patient + family member DOB data
counts[item.key]per-nav small numbers (4 active, 6 seats, 12 events…)
recentActivityexisting activity feed, filtered to last 24–48h, with kind: family|curated|auto on each row for the tag column

Suggested endpoint: GET /api/care-circle-family/home-summary returning all of the above so the page loads in one query.

Step 5

Lift the styles into Tailwind tokens (or a CSS module)

Two acceptable paths — pick the one that matches your codebase preference:

5a. Tailwind extension — your tailwind config already has the warm palette. Add utilities for the new patterns:

// tailwind.config.ts — additions sketch
extend: {
  borderWidth: { '3': '3px' },
  fontFamily: { display: ['Lora', 'Georgia', 'serif'], text: ['EB Garamond', 'Georgia', 'serif'] },
  boxShadow: { paper: '0 14px 32px rgba(35,25,19,0.18)' },
}

Then write the masthead, hero, section-head etc. as Tailwind class strings.

5b. CSS module — copy the <style> block from Care Circle Family.html into family-home.module.css, drop the appbar styles (already provided by your shell), and import per component. Faster path; less DRY than Tailwind.

Either way, the design tokens that already live in app/globals.css (paper, ink, ember, forest) are the source of truth — don't re-define colours.

How to validate

  • Visual diff against Care Circle Family.html at 1280×, 1024×, and 720× widths — the mockup's media queries (980px, 880px, 640px) define the breakpoints.
  • Run the existing tests/e2e/sprint-care-circle-family.spec.ts suite; the nav links and labels are unchanged so most tests should keep passing. Update only the assertions that look for the old "Care & Circle" / "Community" tile structure if any are pinned to those.
  • Lighthouse pass for accessibility — the broadsheet display sizes use clamp(); verify min sizes hit AA contrast on the ember-on-cream foot line.

Reuse with the newsletter

The ProvBadge SVG glyphs and the masthead structure are shared design. When you build the email ellis theme from the previous handoff, use the same SVG paths inside the Jinja prov macro and the same broadsheet rule on the email header. That's the whole point — one identity, one badge set, two surfaces.
